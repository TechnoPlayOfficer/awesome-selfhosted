ACCET Legal Oracle

Institutional Infrastructure for Asset Tokenization & Digital Notarization

1. System Architecture Overview

ACCET Oracle is a distributed verifiability system designed to bridge physical legal assets with the Base L2 blockchain. It utilizes Filecoin for immutable storage and Supabase for high-availability metadata orchestration.

Ingestion Layer Processes PDF documents via Multer, generates SHA-256 hashes, and initiates the Lighthouse SDK upload to Filecoin.
Oracle Layer A Node.js/Express engine that interrogates the Filecoin network to verify CID persistence and Zero-Drift compliance.
Law Layer EVM Smart Contracts (EIP-5192) that bind legal CIDs to on-chain Soulbound Tokens on the Base Network.
Identity Layer Supabase Auth + ERC-3643 conceptual framework for institutional tenant separation and asset holder verification.

2. Technical Data Model

The core schema (Prisma/Postgres) ensures referential integrity between legal tenants and their cryptographic assets.

model Document {
  id          String   @id @default(uuid())

model Tenant {

4. Security & Compliance

A. Storage Immutability

All legal assets are stored on Filecoin. Unlike centralized storage, a CID is content-addressable. If a single byte in the deed changes, the CID changes, triggering an Oracle Alert.

B. Compliance Standards

  • MLETR: Model Law on Electronic Transferable Records compliant metadata structures.
  • ERC-5192: Minimal Soulbound Tokens to prevent unauthorized transfer of legal titles.
  • Audit Logs: Every status change is recorded in Supabase with a cryptographic timestamp.
